# 校园二手书交易平台 

## 1. 项目概述
### 1.1 项目背景
校园内教材、辅导书流转效率低，学生卖书麻烦、买书贵，需要一个轻量化的信息撮合与线上担保支付平台，保障交易安全。

### 1.2 核心目标
- 让学生快速发布/查找二手书信息
- 实现二手书交易核心业务流程（发布→加购→下单→模拟支付）
- 完成前端页面与后端接口的基础联调

---

## 2. 核心功能

### 2.1 用户模块
- 学生身份登录（学号/手机号 + 验证码，验证码固定为「123456」模拟）
- 基础个人信息：昵称、学院、收货地址（仅表单填写/展示，无真实校验）
- 个人中心：我的发布、我的购物车、我的订单

### 2.2 书籍发布模块
- 上传书籍封面/实拍图（本地图片上传，无需对接云存储）
- 填写核心信息：书名、作者、年级、专业、价格、成色（全新/九成新/五成新）
- 发布/编辑/下架操作，发布后展示在首页列表

### 2.3 书籍浏览模块
- 首页展示最新发布书籍列表（封面+书名+价格+成色）
- 支持按「书名/专业」关键词搜索，按「价格」升序/降序排序
- 书籍详情页：展示完整信息 + 「加入购物车」「立即购买」按钮

### 2.4 购物车模块
- 加入购物车：选择书籍后可加入购物车，支持修改购买数量（默认1）
- 购物车列表：展示已加购书籍，可勾选单/多本书籍结算，支持删除操作
- 价格计算：自动计算勾选书籍的总价（无优惠/折扣逻辑）
- 选中状态：勾选书籍后「去结算」按钮可点击，未勾选则置灰

### 2.5 订单与模拟支付模块（无真实支付）
- 结算流程：
  1. 购物车勾选书籍 → 点击「去结算」→ 选择收货地址
  2. 生成订单（订单号规则：时间戳+学号，如 202603161300_2023001）
  3. 模拟支付页：展示订单总价，提供「支付宝支付」「微信支付」按钮（仅点击跳转演示）
  4. 点击支付按钮后，直接跳转「支付成功」页面，更新订单状态为「已支付」
- 订单管理：
  - 订单状态：待支付/已支付/已完成（无发货/收货环节，简化流程）
  - 可查看订单详情，已支付订单可手动标记「已完成」

### 2.6 管理员后台模块
#### 2.6.1 后台登录
- 独立登录入口，管理员账号密码固定（如 admin/123456），无需注册
#### 2.6.2 数据统计面板（核心）
- 核心指标展示（数字卡片形式）：
  - 总用户数、总书籍发布数、总订单数、总交易金额（模拟）
  - 今日新增订单数、今日新增书籍数
- 趋势图表（简化版，用柱状图/折线图展示）：
  - 近7天订单数趋势（无需第三方图表库，可用简单HTML/CSS绘制）
  - 各学院书籍发布数量占比（饼图简化版）
#### 2.6.3 基础管理功能
- 书籍管理：查看所有发布书籍，支持下架违规书籍
- 订单管理：查看所有订单列表，可筛选订单状态（待支付/已支付/已完成）
- 用户管理：查看所有注册用户信息，支持禁用账号

## 4. 非功能需求
- 页面简洁：无需复杂UI设计
- 数据存储：使用本地数据库（SQLite），无需部署到服务器（本地运行即可）
- 支付流程安全合规，遵循支付平台接口规范
